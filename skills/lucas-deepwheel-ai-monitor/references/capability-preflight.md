# 能力体检

## 先确认

- 手机型号、系统和常用方向；
- 只在电脑旁看，还是需要外出访问；
- 只看状态，还是希望批准或控制；
- 使用 Claude Code、Codex CLI、Codex App 或其他 Agent；
- 当前是否已经有结构化的 CURRENT / HANDOFF / status.json。

## 环境检查

```text
文件写入目录：能否生成一个全新输出目录
浏览器：Safari 是否可用，是否需要添加到主屏幕
本机服务：是否允许只监听 127.0.0.1 或局域网
HTTPS：PWA 完整能力是否需要安全来源
状态源：官方状态、结构化文件、已审计适配器是否存在
私人网络：是否已有 Tailscale 等；没有则不要自动安装
通知：是否需要推送；首版默认不需要
```

## 默认最小测试

1. 不接真实数据；
2. 生成假数据 PWA；
3. 在电脑浏览器横向窗口查看；
4. 在 iPhone Safari 同一局域网打开；
5. 检查安全区、横屏、字号、蓝色焦点和自动刷新提示；
6. 用户确认有价值后再接真实状态。

## 降级路线

- 没有状态 API：读取用户主动维护的最小 JSON。
- 没有 HTTPS：先用 Safari 页面验证，不承诺完整 PWA 离线能力。
- 不安装私人网络：仅同一 Wi-Fi 使用。
- 不开放终端：只显示运行、等待、完成和下一步。
