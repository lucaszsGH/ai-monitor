# DeepWheel 手机横屏规范

## 设计目标

让用户在一秒内回答：谁在工作、谁在等我、Context 是否健康、下一步是什么。

## 画布与安全区

- 以手机横屏逻辑尺寸设计，公开版覆盖 812×375（iPhone X 级）到 956×440（iPhone 17 Pro Max 物理分辨率按 3× 投影的逻辑级）范围；具体设备仍需真机复核。
- 内容必须进入 `env(safe-area-inset-*)` 保护区；为覆盖浏览器未返回安全区的场景，左右还要使用 44–62pt 的保守设备护栏。
- 默认使用纯黑底，让刘海／灵动岛融入黑色边界；任何文字、指标和触控目标都不得进入护栏。
- 触控目标不小于 44×44pt。
- 允许系统文本放大，文本容器不得固定高度裁切。

## 布局

- Claude 与 Codex 属于同层级协作主体，概览页采用 1:1 等分双栏，避免视觉暗示一方优先级更高。DeepWheel 的非对称分栏偏好用于主次明确的内容；双 Agent 对等场景登记为产品语义例外。禁止三张等宽卡平铺的硬规则保持不变。
- 顶部为全局目标和同步状态，中部为两 Agent 主区，底部为需要用户处理的唯一行动条。
- 一屏一个蓝焦点；默认把蓝给“需要处理”或当前唯一主行动。
- 运行、等待、错误状态同时使用图标、文字和颜色，不靠颜色单独传意。

## 已确认界面基线

- 私测通过的深色横屏界面是公开 starter 的唯一视觉基线，不再另行重新设计。
- 公开版只允许在该基线上增加双语切换、中性 Agent 名称、通用数据契约和新手引导；不得改变字体层级、颜色语义、双栏比例、会话行节奏和用量结构。
- 字体层级使用 `--type-caption / --type-meta / --type-body / --type-title / --type-count / --type-hero`，按横屏高度自适应。
- 剩余额度颜色固定为：≥70% 充足绿、50–69% 正常蓝、30–49% 偏低橙、<30% 紧急红。颜色与百分比、状态词必须同步变化。
- NOW 只显示两条会话时使用已确认的光学居中校正；SESSIONS 与 NOW 共用同一套字体行高和垂直对齐。
- 任何修改后都必须上述字体、颜色、位置和阈值回归，不能只检查页面是否溢出。

## DeepWheel 视觉合同

```text
主墨：#101722
品牌蓝：#0071E3
雾灰：#F2F5F9
白：#FFFFFF
正文柔灰：#42474F
注释灰：#5C626B
细线：#E1E6EE
成功：#12805C
警告：#B54708
错误：#D92D20
```

公开 starter 必须把这些值登记为 CSS 变量；组件代码不得散写十六进制。废值 `#1D1D1F`、`#1E6FE8`、`#111720`、`#F8FAFC` 一律非法。

当前 iPhone 持续显示场景采用深色投影：纯黑为设备底，`#0B0F15` 为卡片，`#151B24` 为次级面，源品牌值继续以 `--dw-source-*` 登记。反白文字、深色语义色必须单独通过对比度和真机检查，不能把浅色值机械反转。

字体栈：Avenir Next / Inter / PingFang SC / system-ui。字重只使用 400、600、700。

## 双背景

- 本产品默认：纯黑设备底 + 深墨卡片；不使用大面积渐变和发光。
- 黑底覆盖到物理屏幕边缘，卡片和内容停在对称安全区内，使灵动岛视觉隐入边界。
- 卡片只用细线和极弱内高光分层，不使用悬浮白卡或柔投影。

## 动效

- 状态刷新使用低频、克制反馈。
- 尊重 `prefers-reduced-motion`。
- 不使用发光、渐变球、持续脉冲或制造焦虑的倒计时动画。

## 文案

- 手机端短标题、短句；避免长表格。
- 明确标注“模拟 / 本机 / 官方 / 不可用”。
- 不把“同步成功”写成“结论已核实”。
- 对百分比同时写清方向：`Context 已用`、`Quota 剩余`，避免相同数字结构产生相反含义。

## Apple 平台对齐

实现或审查 iPhone 版本时，读取 `references/apple-ios-alignment.md`。Apple 原则用于提升平台适配、可读性和无障碍；不得复制 Apple 受许可限制的设计资源或把 DeepWheel 做成 Apple 外观仿制品。

## 设备依据

- [Apple iPhone 17 Pro / Pro Max specifications](https://www.apple.com/iphone-17-pro/specs/)：17 Pro Max 物理分辨率为 2868×1320，按 3× 逻辑投影对应 956×440 级横屏验证尺寸。
- [WebKit: Designing Websites for iPhone X](https://webkit.org/blog/7929/designing-websites-for-iphone-x/)：从 iPhone X 起使用 `viewport-fit=cover` 与 `env(safe-area-inset-*)` 处理横屏安全区。

## 全屏视口合同

- Safari 浏览器模式使用 `100dvh`，跟随当前可见视口；
- 从主屏幕图标进入的 standalone 模式使用 `100lvh`，覆盖完整横屏高度；
- `viewport-fit=cover` 只负责延伸到物理屏边缘，重要内容仍必须使用 `max(最小护栏, env(safe-area-inset-*))`；
- 必须同时检查页面溢出和 `#app` 底边距视口底边的差值，只检查 `scrollHeight` 不足以发现 iOS 底部留白。
- 公开验收矩阵覆盖 812×375 至 956×440 共 8 类横屏视口，NOW / SESSIONS / USAGE 和 100% / 200% 字体缩放必须通过。
