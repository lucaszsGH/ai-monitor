# 实现选项

## 选择顺序

### 1. 静态样张

- 输入：模板假数据。
- 优点：零凭证、零平台依赖、最快验证横屏价值。
- 降级：始终可用。

### 2. 本机状态 JSON

- 输入：用户或受控脚本写入的最小状态文件。
- 优点：平台稳定、容易审计。
- 风险：状态可能过期，必须显示更新时间。

### 3. 只读适配器

- Claude：优先官方 statusLine / hooks 提供的结构化数据。
- Codex：优先官方状态或经过审计的 JSON 输出。
- 第三方：可参考 Claude HUD、CodexBar、ccusage 等，但先核版本、许可证、权限与输出边界。

### 4. 私人网络

- 只在用户确认后配置。
- 优先私人网络或受认证的 HTTPS；不要直接端口映射到公网。

### 5. 远程批准或控制

- 与只读副屏分离。
- 单独身份验证、最小权限、动作确认和审计记录。
- 首版不做任意命令输入。

## 底座选择

- 想快速试用成品：可评估 MobileCLI 等移动客户端。
- 想深度定制：优先轻量 PWA，不直接背入完整远程 IDE。
- 借开源代码前检查许可证；AGPL 项目要评估网络服务修改后的源码义务。
